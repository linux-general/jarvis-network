# Jarvis Network model library

A mirror of every open-weight model the Jarvis stack uses, hosted on Vultr
Object Storage (ATL). Containers pull from here at boot instead of from
HuggingFace directly — faster, no rate limits, no gated-repo surprises.

## What's in here

- [`manifest.yaml`](manifest.yaml) — declarative catalog. Single source of
  truth. `id`, `hf_repo`, `prefix`, `role`, `size_gb`, `status`.
- A short `README.md` (this file).

The actual model files live in the **`jarvis-models`** bucket on Vultr Object
Storage, region ATL. Container code never edits the manifest at runtime; it
only reads it.

## Workflow

1. **Add a model** — edit `manifest.yaml`, append an entry with `status:
   aspirational`. Commit.
2. **Mirror it** — run `scripts/seed-models.py --only <id>` from a Vultr ATL
   instance (the seed script is built for HF → Vultr OS streaming over Vultr's
   gigabit backbone; running it from a residential connection is wasteful).
3. **Promote** — flip `status: current` once the bucket copy passes
   `--verify`. Commit.
4. **Containers pull** — `seed-models.py --pull <id> --dest /opt/models/<id>`
   inside the container's entrypoint.

## Why we mirror instead of pulling from HF

- HF rate-limits anonymous downloads hard during peak hours.
- Several models we use are gated (`pyannote/*`, some Llama variants) — one
  HF_TOKEN handles the mirror once, then every container pulls credential-free.
- Vultr OS ATL → Vultr instance ATL is free intra-DC bandwidth.
- HF can take a repo down; our archive doesn't disappear.

## Adding a new model: checklist

- [ ] Verify the exact `hf_repo` id resolves on huggingface.co
- [ ] Check the license — only redistribute Apache-2.0, MIT, OpenRAIL, or
      Llama 3.x / Qwen / DeepSeek community-style licenses
- [ ] Estimate `size_gb` (`huggingface-cli scan-cache` if it's local)
- [ ] Set `status: aspirational` until first mirror passes verify
- [ ] Commit the manifest change BEFORE running `seed-models.py` (so the bucket
      contents match the manifest in git history)

## What we deliberately do NOT mirror

- **DeepSeek-V3 / V3.2** — 671B parameters, ~700 GB each. Outside the bucket
  budget. Served via OpenRouter cloud fallback when needed.
- **Closed-weight or signed-EULA models** — Gemini, GPT, Claude (obvious).
- **Models we'd never run** — base/foundation models without instruct
  fine-tunes (no use case here).

## Bucket access

- Read: signed URLs, 1-hour expiry, generated by `seed-models.py --signed-url <id>`.
- Write: only from machines holding the seed access key (stored in 1Password
  vault entry **"Vultr OS – jarvis-models writer"**, never in git).
